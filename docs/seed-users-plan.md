# NewClaw 种子用户获取计划：0 -> 100

> 目标：30 天内获取 100 名活跃种子用户，验证产品-市场匹配度
> 衡量标准：Star + Clone + 至少运行过一次 `pnpm dev` 启动 Trinity 循环

---

## 1. 种子用户画像

### Tier 1 -- 高转化（目标 50 人）

**AI Agent 框架不满者**
- 用过 LangChain/AutoGPT/CrewAI，踩过坑：Agent 跑飞烧 token、无权限控制、输出不可审计
- GitHub profile 有 AI 相关 repo，Star 过竞品框架
- 搜索行为：`AI agent framework governance`、`LangChain alternative`、`autonomous agent safety`
- 切入话术：你的 Agent 跑飞过几次？NewClaw 用三权分立 + L0-L4 保险丝矩阵解决这个问题

**独立开发者 / 创客**
- 活跃在 HackerNews、Reddit、V2EX 的 Show HN 型开发者
- 喜欢尝新，愿意为一个有趣的架构理念投入周末时间
- 切入话术：5 分钟跑通第一个 AI 三核辩论循环，不需要 API Key

### Tier 2 -- 中等转化（目标 30 人）

**Web3 x AI 交叉研究者**
- 关注 DID、DAO 治理、链上 AI Agent 话题
- 活跃在 Mirror.xyz、Farcaster、Telegram Web3 群
- 切入话术：NewClaw 的 Agent 有 DID 身份 + New.B 经济系统，这不是套壳 DAO，是 AI 原生治理

### Tier 3 -- 扩散节点（目标 20 人）

**AI Agent 领域 KOL / 内容创作者**
- Twitter/X 粉丝 1K-50K 的 AI 技术博主（非泛科技大 V，太大不精准）
- 掘金/知乎的 AI Agent 专栏作者
- 切入话术：给你独家 demo 权限 + Genesis Node 标识，换一篇深度体验文

## 4. 种子用户激励体系

### Genesis Node 永久标识
- 前 100 名用户获得 `Genesis Node` 身份标签，写入链上 DID 元数据
- 这个标识永久不可复制，后来者无法获得
- 价值锚点：当网络扩展到 10 万节点时，Genesis Node 是创世成员的不可篡改证明

### New.B 创世奖励（递减结构）

| 阶段 | 用户编号 | 注册奖励 | 推荐奖励（每人） |
|------|---------|---------|----------------|
| 创世 | #1-#25 | 200 New.B | 50 New.B |
| 先锋 | #26-#50 | 100 New.B | 30 New.B |
| 早期 | #51-#100 | 50 New.B | 20 New.B |
| 常规 | #101+ | 10 New.B | 5 New.B |

核心逻辑：20 倍的奖励差距让前 25 人有极强的稀缺感和传播动力。

### 核心群特权
- 前 100 名进入 Discord `#genesis` 私密频道
- 产品路线图投票权重 x3（vs 普通用户 x1）
- 每周一次与创始团队的 30 分钟闭门会

## 5. 反馈收集机制

### 三层反馈漏斗

**Layer 1 -- 被动收集（全自动）**
- GitHub Issue 模板：Bug Report / Feature Request / 上手体验反馈（3 个模板）
- 应用内：首次启动 Trinity 循环成功后弹出 1 题 NPS（0-10 分）
- 自动采集：`pnpm dev` 启动成功/失败率、首次运行到第一个 Trinity 循环完成耗时

**Layer 2 -- 主动收集（半自动）**
- Day 15 + Day 30 各发一次结构化问卷（Tally/Typeform），5 题以内：
  1. 你之前用什么 AI Agent 框架？遇到什么问题？
  2. NewClaw 解决了你的什么痛点？（多选）
  3. 你最想要但目前没有的功能是什么？（开放题）
  4. 你会向同事推荐 NewClaw 吗？（NPS 0-10）
  5. 你愿意参加 15 分钟用户访谈吗？（留联系方式）

**Layer 3 -- 深度访谈（纯手动）**
- 从问卷 Q5 中筛选愿意访谈的用户，优先选：
  - NPS 9-10 的（了解为什么爱，提炼卖点）
  - NPS 0-6 的（了解为什么不满意，定位阻塞点）
- 每周 3-5 场 15 分钟 1v1 视频通话
- 固定问题框架：
  - 你第一次启动时卡在哪一步？（上手摩擦）
  - 如果没有 NewClaw，你用什么替代方案？（竞品锚定）
  - 你愿意为哪个功能付费？（变现信号）

### 反馈闭环规则

- 每个 GitHub Issue 48 小时内必须响应（即使只是 acknowledged）
- 每周五在 Discord #changelog 发布本周改进清单，明确标注来源 Issue
- 用户反馈直接导致的改进，在 Changelog 中 @提到 该用户

## 6. 关键里程碑与决策点

| 里程碑 | 时间 | 指标 | 未达标行动 |
|--------|------|------|-----------|
| 首个外部 Star | Day 1 | >= 1 非团队 Star | 检查 README 是否有阻塞 |
| HN 首页 | Day 2-3 | 前 30 排名 | 调整标题和 Show HN 格式 |
| 10 个活跃用户 | Day 7 | 10 人跑通 Trinity | 若 < 5 人，说明上手门槛太高，紧急修 Quick Start |
| 50 个用户 | Day 14 | 50 人注册 | 若 < 25 人，暂停中文社区，加大英文社区投入 |
| 100 个用户 | Day 30 | 100 人注册 | 若 < 70 人，延长至 Day 45，不启动 ProductHunt |
| NPS >= 40 | Day 30 | 问卷结果 | 若 < 30，暂停增长，聚焦产品改进 |

> 终极判断标准：如果 Day 30 时 NPS < 30 且活跃用户 < 50，说明产品未达到 PMF，
> 应暂停一切增长动作，用 2-4 周密集迭代产品后再重启获客。
