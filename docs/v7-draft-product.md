# NewClaw v7.0 产品需求规格书

**状态**: 草案 | **作者**: 产品经理 Alex | **日期**: 2026-04-10 | **版本**: 0.1
**输入文档**: v6.0白皮书 / B队资产清单 / A队推进计划B

---

## 0. 核心判断：v6.0 为什么无法直接编码

v6.0 白皮书定义了愿景（百万节点蜂群、New.B 通缩货币、联邦防御网），但未回答三个关键工程问题：
(1) 用户在屏幕上看到什么？(2) 用户点击什么完成任务？(3) 每个操作的成功/失败如何判定？
v7.0 的职责是将白皮书翻译为**可验收的需求**。B队已实现 88% 的引擎逻辑（7模块/188测试），
v7.0 聚焦于将模拟桩替换为真实执行，并补齐身份、沙盒、AI编排三块缺口。

## 1. 用户故事地图

### 角色定义

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 节点运营者 | 部署并运行 Trinity 节点的个人/团队 | 一键启动、可视化监控、收益可见 |
| 开发者 | 基于 NewClaw 构建应用或扩展模块的工程师 | API 清晰、文档完整、可调试 |
| 宿主 | 提供算力资源并获取分红的基础设施提供者 | 成本透明、分红自动、风险可控 |

### 1.1 节点运营者故事

| 编号 | 用户故事 | 验收标准 |
|------|----------|----------|
| U-01 | 作为节点运营者，我希望一键生成节点身份密钥对，以便节点拥有全局唯一的不可伪造身份 | AC1: 点击"创建节点"后 <2s 生成 Ed25519 密钥对；AC2: 公钥派生的 DID 格式为 `did:newclaw:<hex32>`；AC3: 私钥仅存储在本地加密 keystore 中，不可通过 UI 或 API 导出明文 |
| U-02 | 作为节点运营者，我希望看到 Trinity 三核引擎的实时状态面板，以便了解 AI-1/AI-2/AI-3 各自在做什么 | AC1: 面板显示三个代理卡片，每个含状态（idle/thinking/blocked）、当前任务、累计完成数；AC2: 状态变更在 <500ms 内反映到 UI；AC3: 展开卡片可查看该代理的全部输出历史 |
| U-03 | 作为节点运营者，我希望创建一个任务并观看它走完 proposal->audit->approval->execution->settled 全流程，以便验证三体治理在正常工作 | AC1: 输入任务标题和描述后，任务自动进入 AI-1 提案阶段；AC2: 每个阶段转换产生系统事件且在事件流中可见；AC3: 任务完成后自动生成 OutcomeReport 并写入 Oracle |
| U-04 | 作为节点运营者，我希望查看治理账本（六本账）的内容，以便审计节点的全部决策记录 | AC1: 账本页面支持六个标签页（证据/价值/债务/时效/判例/交易）切换；AC2: 每个标签页显示条目列表，支持按时间排序；AC3: 证据条目显示 H1-H4 等级标识并支持按等级筛选 |
| U-05 | 作为节点运营者，我希望看到节点的晋升进度（Stage 0-4），以便知道还需要完成什么才能升级 | AC1: 晋升页面显示当前阶段及四维进度条（outcomes/compliance/reconciliation/stability）；AC2: 显示下一阶段的门槛值与当前差距；AC3: 满足条件时显示"可晋升"按钮 |
| U-06 | 作为节点运营者，我希望在知识市场浏览和购买其他节点的 Playbook，以便复用成功经验 | AC1: 市场页面列出所有在售 Playbook，含价格、分类、卖家信用分；AC2: 购买操作扣除买方余额、增加卖方余额、生成双向账本记录；AC3: 余额不足时显示错误提示而非静默失败 |
| U-07 | 作为节点运营者，我希望高风险操作需要人工确认，以便防止 AI 自主执行危险动作 | AC1: L3 级操作弹出确认对话框，含操作描述和风险说明；AC2: L4 级操作需要人工+AI-3 双签才可执行；AC3: 未获批准的请求状态为 denied，不执行任何后续动作 |

## 3. MVP 核心流程：从安装到完成第一个 Trinity 循环

```
用户动作                        系统行为                               对应模块
--------                        --------                               --------
[1] 安装并启动应用
    pnpm install && pnpm dev    Tauri 窗口启动，加载主界面               Tauri + Vite
         |
         v
[2] 首次初始化
    系统检测无节点身份            弹出"创建节点"引导页                    UI / router
         |
         v
[3] 创建节点身份
    点击"生成密钥对"             Ed25519 密钥对生成 -> 私钥加密存储       identity 模块 (A队新建)
                                -> DID 派生 -> 写入 V6SystemState
                                -> 初始化默认宪法(Constitution)           constitution.ts
                                -> 初始化空白六本账(Ledgers)              ledger.ts
                                -> 初始化 SIM 1000 余额                  market.ts
                                -> 创建三核代理(Trinity Agents)           engine.ts
                                -> 设置节点阶段 Stage-0                  promotion.ts
         |
         v
[4] 进入主仪表盘
    看到 Dashboard 视图          显示三核面板(idle) + 空任务列表           Trinity/index.tsx
                                + 事件流(显示: 节点创建成功)              DashboardView
         |
         v
[5] 创建第一个任务
    填写标题+描述，点击"创建"    createNewTask(title, desc)               store.createNewTask
                                -> 任务状态: draft -> proposal            engine.createTask
                                -> AI-1 状态: idle -> thinking             engine.updateAgentStatus
                                -> 事件: task:created                      V6Event
         |
         v
[6] AI-1 提交提案
    (自动/手动触发)              submitProposal(taskId, content)          store.submitProposal
                                -> 任务阶段: proposal -> audit            engine.runProposalPhase
                                -> 生成 TrinityOutput(task-draft)
                                -> AI-1: thinking -> idle
                                -> AI-2: idle -> thinking
                                -> 事件: trinity:output
         |
         v
[7] AI-2 提交审计
    (自动/手动触发)              submitAudit(taskId, findings, risk)      store.submitAudit
                                -> 检查权限级别 vs 保险丝矩阵             fuse-matrix.ts
                                -> 若 risk=critical -> 任务 blocked
                                -> 若通过 -> 阶段: audit -> approval
                                -> 生成 TrinityOutput(audit-opinion)
                                -> AI-2: thinking -> idle
                                -> AI-3: idle -> thinking
         |
         v
[8] AI-3 审批决策
    (自动/手动触发,              submitApproval(taskId, approved, budget)  store.submitApproval
     L3+需人工确认弹窗)          -> 若 L4: 检查是否有 human 双签
                                -> 若批准 -> 阶段: approval -> execution
                                -> 写入价值评估到 VALUE 账本              ledger.addValueAssessment
                                -> 事件: task:approved
         |
         v
[9] 执行任务
    (Docker沙盒 或 模拟)         沙盒执行代码/模拟执行                    sandbox (A队) / setTimeout
                                -> 获取执行结果 + evidenceGrade
         |
         v
[10] 完成与结算
    (自动触发)                   completeTask(taskId, result, grade)      store.completeTask
                                -> 阶段: execution -> settled
                                -> 生成 OutcomeReport                     oracle.generateOutcomeReport
                                -> 规则评估 -> 结算                       oracle.evaluateOutcome
                                -> 写入证据到 EVIDENCE 账本               ledger.addEvidence
                                -> 写入判例到 CASELAW 账本                ledger.addCaseLaw
                                -> 写入交易到 LOCAL 账本(+奖励)           ledger.addLocalLedgerEntry
                                -> 更新节点指标                           promotion.calculateNodeMetrics
                                -> 事件: outcome:settled
         |
         v
[11] 验证结果
    用户查看各页面               仪表盘: 任务显示 settled 状态
                                账本页: 新增证据/价值/交易条目
                                Oracle: 新增结算报告
                                节点: 晋升进度更新
                                -- 第一个 Trinity 循环完成 --
```

**MVP 成功标准**: 用户从 `pnpm dev` 到看到第一个任务 settled 状态，全程 < 5 分钟，零手动配置文件编辑。

## 4. 非功能需求

### 4.1 性能指标

| 指标 | 目标值 | 测量方法 | 优先级 |
|------|--------|----------|--------|
| 应用启动时间（冷启动） | < 3s | 从进程启动到主界面可交互 | P0 |
| IPC 调用响应时间（本地操作） | < 100ms (p95) | 单次 IPC roundtrip | P0 |
| 任务阶段推进延迟 | < 500ms | 从触发到 UI 状态更新 | P0 |
| 单元测试全量执行时间 | < 30s | `pnpm test` 总耗时 | P1 |
| 内存占用（空闲状态） | < 200MB | Tauri 进程 RSS | P1 |
| 内存占用（100任务并发） | < 500MB | Tauri 进程 RSS | P2 |
| Docker 沙盒启动时间 | < 5s | 从调用到容器 ready | P1 |
| 沙盒代码执行超时 | 30s（可配置） | 硬性截止时间 | P0 |
| 账本查询响应（10000条记录） | < 200ms | 分页查询返回时间 | P2 |

### 4.2 安全要求

| 要求 | 描述 | 验收标准 | 优先级 |
|------|------|----------|--------|
| 私钥保护 | 节点私钥必须加密存储，不可通过任何接口导出明文 | 静态分析确认无明文私钥日志/序列化路径；keystore 使用 AES-256-GCM 加密 | P0 |
| 沙盒隔离 | Docker 容器无宿主文件系统访问、无网络（默认） | 容器配置验证: `--network none`、无 volume mount（除输入/输出目录） | P0 |
| 权限分级 | L3/L4 操作必须经过人工确认才可执行 | 无人工确认时 L3/L4 操作被拦截，拦截事件写入判例账本 | P0 |
| 输入验证 | 所有用户输入经过校验和转义 | 任务标题/描述长度限制（标题<200字、描述<5000字）；无 XSS 可注入路径 | P1 |
| 数据完整性 | 账本数据不可被外部篡改 | 每次写入账本生成 SHA-256 哈希链，启动时校验完整性 | P1 |

### 4.3 可用性目标

| 要求 | 描述 | 验收标准 | 优先级 |
|------|------|----------|--------|
| 首次使用引导 | 新用户无需阅读文档即可完成第一个 Trinity 循环 | 5 名测试用户中 4 人在 10 分钟内完成，无求助 | P0 |
| 错误信息可读 | 所有错误提示使用用户可理解的语言 | 无裸露的技术堆栈/error code；每个错误提示含"发生了什么"+"建议操作" | P1 |
| 状态可恢复 | 应用崩溃后重启可恢复到崩溃前状态 | Zustand persist 保证状态持久化；重启后任务列表/账本/余额完整 | P0 |
| 响应式反馈 | 每个用户操作在 200ms 内有视觉反馈 | 按钮点击有 loading 状态；长操作有进度指示 | P1 |

### 4.4 兼容性要求

## 5. 竞品差异点：NewClaw vs 传统 Agent 框架

| 维度 | LangChain | AutoGPT | CrewAI | **NewClaw v7.0** |
|------|-----------|---------|--------|-------------------|
| **核心模型** | 链式调用（Chain） | 单 Agent 自主循环 | 多角色协作 | **三权分立治理（提案/审计/决策）** |
| **治理机制** | 无 | 无 | 角色分工但无制衡 | **宪法约束 + 五级权限保险丝 + 人工双签** |
| **决策审计** | 无内置 | 有日志但无结构化 | 无 | **六本治理账本 + H1-H4 证据等级 + 判例法** |
| **执行安全** | 依赖用户实现 | 无沙盒 | 无沙盒 | **Docker 隔离沙盒 + 资源限制 + 超时熔断** |
| **结果验证** | 无 | 无 | 无 | **Oracle 预言机 + PoO 结果证明 + 自动结算** |
| **经济激励** | 无 | 无 | 无 | **New.B 模拟货币 + 知识市场交易 + 质押惩罚** |
| **进化路径** | 无 | 无 | 无 | **五阶段晋升流水线（模拟->联邦），四维评分门槛** |
| **风控能力** | 无 | 弱（预算限制） | 无 | **AI-2 冷酷审计 + 风险熔断 + 宪法硬约束** |

**一句话差异**: LangChain/AutoGPT/CrewAI 让 AI 做事；NewClaw 让 AI **在治理框架内做正确的事并证明它做到了**。

### 竞品未解决、NewClaw 已回答的三个问题

1. **"AI 做了什么决策？依据是什么？"** -- 六本治理账本 + 证据等级，每个决策可追溯、可审计。
2. **"如何防止 AI 自嗨空转？"** -- PoO 结果验证 + PriorityScore >= 85 门槛，不达标则质押扣除。
3. **"AI 的经验如何积累和传递？"** -- Playbook 知识市场，成功经验定价交易，PoO 追踪买家使用结果。
