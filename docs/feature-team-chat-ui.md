# 功能需求：团队群聊工作台

> 来源：用户产品灵感 2026-04-10
> 优先级：P1（核心体验差异化）
> 状态：需求确认，待排期

## 一、需求背景

当前 Trinity 界面是单一面板视图，用户无法直观看到三个 AI 代理之间的互动过程。
参考竞品（newclaw-grok 的群聊界面），多 Agent 群聊窗口能显著提升用户体验和信任感。

## 二、核心功能点

### F1: 群聊窗口布局
- 类似即时通讯的聊天气泡界面
- 每个 AI 代理有独立的头像、名称、颜色标识
- AI-1（扩张者）蓝色气泡，AI-2（审计者）琥珀色气泡，AI-3（治理者）翠绿色气泡
- 用户消息居右，AI 消息居左（按角色区分）
- 支持查看完整的 proposal → audit → approval 对话链

### F2: 实时工作状态动画
- AI 思考/执行时显示动态加载动画（如雷达扫描旋转效果）
- 让用户明确知道"AI 正在工作"而非"卡住了"
- 不同状态有不同动画：思考中（旋转）、执行中（脉冲）、完成（勾号动画）
- 验收标准：用户在 AI 处理期间不会怀疑系统停工

### F3: 消息元数据显示
- 每条消息显示：日期时间戳
- Token 使用信息（in/out/cache）
- 消息操作按钮：复制、引用、展开/折叠

### F4: 多 Agent 可视化互动
- 当 AI-1 提案时，消息实时出现在群聊中
- AI-2 审计意见作为"回复"出现，视觉上关联到被审计的提案
- AI-3 批准/拒绝作为最终裁决，视觉上有明确的"决策"标记
- 用户可以在任意环节插入意见或指令

## 三、技术方案概要

### 方案 A：在现有 Trinity 页面新增 "群聊" 视图标签
- 新增 V6View: 'team-chat'
- 复用现有 TrinityTask.outputs 数据，按时间排列为聊天气泡
- 最小改动，快速交付

### 方案 B：独立的群聊页面组件
- 全新的 ChatBubble、AgentAvatar、MessageMeta 组件
- 独立的消息流状态管理
- 更灵活，但工作量大

### 建议：方案 A 先行（MVP），验证用户价值后再升级为方案 B

## 四、设计参考

- 竞品参考：newclaw-grok 群聊界面
- 气泡样式参考：现有 Chat 页面的 ChatMessage 组件
- 动画参考：雷达扫描旋转效果（CSS animation）
- 元数据参考：token 计数 + 时间戳 + 操作按钮

## 五、验收标准

- [ ] 用户能在一个聊天窗口中看到 AI-1/AI-2/AI-3 的完整对话过程
- [ ] 每条消息有时间戳、token 信息、复制/引用按钮
- [ ] AI 工作时有明确的动态加载动画
- [ ] 对话链清晰展示 proposal → audit → approval → result 流程
- [ ] 支持用户在群聊中插入指令
